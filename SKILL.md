# 舞蹈剪辑 Skill

## 触发条件

- 用户给出一段舞蹈视频并要求"按模版剪"
- 用户提到"舞蹈卡点""舞蹈锁点""关键帧锁点""dance edit""dance clip""剪一段舞蹈"
- 用户引用本 skill:`@dance-edit` / `使用 dance-edit`

不触发:用户只想看动作分析、只想提取某一帧、只想转格式、只想压缩。

## 输入(必填 + 选填)

| 字段 | 必填 | 说明 |
|---|---|---|
| `source_video` | ✅ | 源文件绝对路径(`.mp4`/`.mov`) |
| `output_path` |  | 默认 `source_video` 同目录 `<name>_edit_<ts>.mp4` |
| `duration` |  | 成片时长(秒),**默认 = 源长**。不切,BGM 完整保留。要裁短就显式指定。 |
| `add_subtitle` |  | 默认 `True`,沿用模板下沿白字风格 |
| `bpm_override` |  | 用户指定节拍时用,跳过自动检测 |
| `color_grade` |  | `blue_orange`(默认)/ `none` |

如果用户没指定时长,**直接用源长**,不要默认截断。要裁短就显式指定,不要靠"经验推荐"截。

## 核心技法(从模版抽出来的)

模版原片是教学视频,但实际产出的"舞蹈成片"效果只有 4 个动作,严格复刻这 4 步:

1. **锚点锁帧(Anchor Lock)** — 用 mediapipe pose 追踪每帧舞者的两个关键点:
   - 红色 **+** 锁 **胸口**(mediapipe landmark 11/12 中点)
   - 黄色 **−** 锁 **脚尖**(landmark 27/28 中点)
2. **背景反向运动** — 每帧用仿射变换把"胸口锚点"贴回画面固定位置(画面中点偏上 1/3),背景自然反向滑动,形成"舞者悬浮、场景流淌"效果。
3. **节拍卡点(Beat Sync)** — 用 librosa 测 BPM,在节拍位置做 1–2 个 hard cut(切不同景别或不同灯光段落),其余保持连续。
   - **决策规则**:如果源素材是**单一连续景别**(全程同一机位/灯光/构图,**没有镜头切换、没有灯光变化**),**跳过 hard cut**。在单镜头里自动切会变成"卡碟/定格重启"的故障感,不是舞蹈视频想要的。
   - 只在源素材包含 ≥2 个明显不同段落(开场/副歌/灯光切换/不同机位)时才做 hard cut,且每个 cut 必须落在节拍位置上。
4. **蓝橙调色(Blue-Orange)** — 阴影往青蓝偏,高光往橙黄偏,对比度 +15,saturation +20。模版里这是"网红色",不要省。

## 处理流程

### Step 1. 探针分析(必做,先报告后动手)

跑 `scripts/analyze.py`,输出 JSON 报告:
- 分辨率、fps、总时长、编码
- 舞者 bbox 占比(若 < 30% 提示用户镜头太远)
- 音频 BPM(给一个建议)
- 推荐的 `start_time` / `duration`(选舞者最居中、最有动作的连续段)
- 每帧的胸口 + 脚尖 anchor 坐标(缓存到 `<work>/anchors.json`)

把这份报告**先回给用户**,关键数字(建议时长、建议起始点、BPM)列出来,让用户确认或修改。**不要直接开剪。**

### Step 2. 生成 ffmpeg 关键帧

每 5 帧采样 1 个 zoompan keyframe,把 anchor 位置 → ffmpeg `zoompan` 表达式:
- 目标锚点:画面 (0.5, 0.35) 居中偏上
- 缩放:基于 anchor Y 位置自适应(脚尖越低,zoom 越紧)

### Step 3. 应用画面变换 + 调色

ffmpeg pipeline:

```
[0] input source
[1] zoompan 按 keyframes 锚定胸口到 (0.5, 0.35)
[2] scale 到 1080x1920(9:16 竖屏)
[3] eq=contrast=1.15:saturation=1.20
[4] colorbalance bs=0.15:bm=0.10:hm=0.15:rm=0.05(蓝橙调色)
[5] drawtext 字幕(可选,黑底白字+描边,底部居中)
[6] h264 + aac 输出
```

音频策略:截取对应 `start_time` 段的原音频,**不重混 BGM**。原片自带音乐最稳。

### Step 4. 验收

输出后**必看一帧**(`ffmpeg -ss 1 -frames:v 1`),确认:
- 舞者是否居中(胸口应在画面 1/3 高度位置)
- 背景是否有"反向流动"感
- 调色蓝橙是否到位
- 字幕是否遮挡面部

如有问题,回到 Step 2 调关键帧密度 / Step 3 调 color balance,不要重做 Step 1。

## 输出规范

- 容器:MP4
- 视频:H.264, yuv420p, 30fps, 1080×1920(9:16)
- 音频:AAC 128kbps(沿用源音频,不改速)
- 码率:视频 6 Mbps(短视频标准)
- 文件名:`<原名>_edit_<YYYYMMDD-HHMMSS>.mp4`

## 反例 / 不要做

- ❌ 不要在锚点缺失帧硬填 0,会跳变。必须插值。
- ❌ 不要把整段视频一次性 zoompan,会变幻灯片。**只在选中的 12–30s 段做。**
- ❌ 不要加炫技转场(旋焦、glitch、粒子)。模版里就是硬切 + 字幕驱动,模仿它。
- ❌ 不要做 1:1 横屏输出。9:16 是硬要求。
- ❌ 不要"智能补帧"或插值升帧,会糊。30fps 起步就行。
- ❌ 字幕不要盖脸。垂直位置 y=0.85(画面下沿),字号 ≤ 48px(1080p 下)。

## 排错快查

| 症状 | 原因 | 修法 |
|---|---|---|
| 舞者"漂移"出画面 | 锚点采样太稀 | 关键帧间隔从 5f 改 2f |
| 背景"果冻" | zoompan step 太大 | 改用 crop+scale 表达式 |
| 追踪丢失(整段) | mediapipe 找不到人 | 提示用户镜头太偏 / 加全图裁剪再试 |
| 蓝橙不"网感" | 颜色 balance 给太少 | colorbalance bs=0.15,bm=0.10, hm=0.15 |
| 字幕抖动 | drawtext 用绝对坐标 | 改用 `enable='between(t,...)'` 锁时间窗 |
